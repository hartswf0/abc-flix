Operations Manual & Philosophy

The Operational Paradigm

"I want to create visual ambiguity, something that demonstrates that recognizable images... can be made while observing them under severe or unusual constraints. In spite of such constraints, we're successful at giving meaning to what we see... My greatest fun comes from using odd building blocks."

— Ken Knowlton (Masters of Deception)

Execution Rule: Do not attempt to draw reality. Assemble "odd building blocks" (integer values on a 64x48 grid). The constraints are absolute; the blurring mechanism forces your visual cortex to assign meaning to the resulting ambiguity.

Protocol 1: The Grid Constraints

  • The output target is an absolute matrix: 64 X-columns by 48 Y-rows.
  • Origin point (X:0, Y:0) is the top-left corner.
  • Intensity constraints: Input must be an integer between 0 (White) and 7 (Black).
  • Line length constraint: Lines exceeding 80 characters result in fatal compilation failure.

Protocol 2: Command Syntax

Command Format Execution Result
CLR v Fills all grid coordinates with intensity v.
PNT x y w h v Draws a block starting at x,y, spanning width w and height h, applying intensity v.
LIN x1 y1 x2 y2 v Plots a trajectory from x1,y1 to x2,y2 with intensity v.
REC n Writes the current array to the buffer n times.
SHF dx dy n Translates the entire matrix by dx, dy units per frame, looping n times.

Protocol 3: Output Extraction
